# 14.2.1: Wildcard Mask Overview

#### Core Concept
*   **Wildcard Mask:** A 32-bit mask used in ACLs and OSPF to specify which bits of an IPv4 address must match and which can be ignored.
*   It's the **inverse** of a subnet mask.

#### The Golden Rules
The logic is simple and opposite to a subnet mask:
*   **Wildcard bit `0`**: **Match** the corresponding bit in the address.
*   **Wildcard bit `1`**: **Ignore** the corresponding bit in the address.

---

#### Common Examples & Keywords

| Wildcard Mask         | Binary (Last Octet) | Meaning                                          | Common Use                                                      |
| :-------------------- | :------------------ | :----------------------------------------------- | :-------------------------------------------------------------- |
| **`0.0.0.0`**         | `00000000`          | Match all 32 bits exactly.                       | Match a single host (e.g., `host 10.1.1.1`)                     |
| **`0.0.0.255`**       | `11111111`          | Match the first 3 octets, ignore the last octet. | Match all hosts in a /24 subnet.                                |
| **`0.0.0.15`**        | `00001111`          | Match first 28 bits, ignore the last 4 bits.     | Match a specific range of addresses (e.g., x.x.x.0 to x.x.x.15) |
| **`255.255.255.255`** | N/A                 | Ignore all 32 bits.                              | Match any address (keyword `any`)                               |

# 14.2.2: Wildcard Mask Use Cases

#### 1. Matching a Specific Host
To create an ACL rule that applies to only one specific device.

*   **Goal:** Match the IP address `192.168.1.1` exactly.
*   **Wildcard Mask:** `0.0.0.0`
*   **Logic:** Every bit `0` in the mask forces a match. All 32 bits must be identical.
*   **Command:** `access-list 10 permit 192.168.1.1 0.0.0.0`

---

#### 2. Matching an IPv4 Subnet
To create a rule that applies to all devices within a single subnet.

*   **Goal:** Match all hosts in the `192.168.1.0/24` network.
*   **Wildcard Mask:** `0.0.0.255`
*   **Logic:**
    *   The first three `0`s mean the first three octets must match `192.168.1`.
    *   The final `255` (binary `11111111`) means the last octet is ignored.
*   **Command:** `access-list 10 permit 192.168.1.0 0.0.0.255`

---

#### 3. Matching a Range of IPv4 Addresses
To create a single rule that covers multiple subnets (summarization).

*   **Goal:** Match a range of networks, for example, from `192.168.16.0/24` to `192.168.31.0/24`.
*   **Wildcard Mask:** `0.0.15.255` (for the example address `192.168.16.0`)
*   **Logic:** This is more advanced. It works by matching the bits that are common to all networks in the range and ignoring the bits that change.
    *   The first two `0`s match `192.168`.
    *   The `15` (binary `00001111`) in the third octet tells the router to match the first 4 bits but ignore the last 4. This covers the range from 16 to 31.
    *   The final `255` ignores the entire host portion.
*   **Command:** `access-list 10 permit 192.168.16.0 0.0.15.255`

---

#### Quick Reference Keywords

| To Match... | Wildcard Mask | Keyword | Example |
| :--- | :--- | :--- | :--- |
| A single host | `0.0.0.0` | `host` | `permit host 192.168.1.1` |
| Any address | `255.255.255.255` | `any` | `permit any` |

# 14.2.3: Wildcard Mask Calculation

#### The Key Formula (Shortcut Method)
The easiest way to calculate a wildcard mask is to subtract the subnet mask from `255.255.255.255`.

```
  255.255.255.255
- [Your Subnet Mask]
-------------------
= [Resulting Wildcard Mask]
```

---

#### Practical Examples

| Goal                                      | Subnet Mask         | Calculation                              | Resulting Wildcard Mask |
| ----------------------------------------- | ------------------- | ---------------------------------------- | ----------------------- |
| **Match a /24 network**<br>(e.g., `192.168.3.0/24`) | `255.255.255.0`     | `255 - 0 = 255`                          | `0.0.0.255`             |
| **Match a /28 network**<br>(e.g., `192.168.3.32/28`) | `255.255.255.240`   | `255 - 240 = 15`                         | `0.0.0.15`              |
| **Summarize two networks /23**<br>(e.g., `192.168.10.0/23`) | `255.255.254.0`     | `255 - 254 = 1`, `255 - 0 = 255`       | `0.0.1.255`             |
| **Summarize a range /20**<br>(e.g., `192.168.16.0/20`) | `255.255.240.0`     | `255 - 240 = 15`, `255 - 0 = 255`      | `0.0.15.255`            |

# 14.2.4: Wildcard Mask Keywords

#### Purpose
To simplify ACL configuration and improve readability, Cisco IOS provides two keywords that replace the most common wildcard masks.

**Benefits:**
*   Fewer keystrokes.
*   Makes the intent of the rule much clearer.

---

#### The Keywords

| Keyword | Replaces Wildcard Mask | Meaning                                   |
| :------ | :--------------------- | :---------------------------------------- |
| `host`  | `0.0.0.0`              | Matches a single, specific host address.  |
| `any`   | `255.255.255.255`      | Matches any IP address.                   |

---

#### Examples of Usage

**1. To permit a single host:**

*   **Manual Method:**
    `R1(config)# access-list 10 permit 192.168.10.10 0.0.0.0`
*   **Keyword Method (Preferred):**
    `R1(config)# access-list 10 permit host 192.168.10.10`

**2. To permit any host:**

*   **Manual Method:**
    `R1(config)# access-list 11 permit 0.0.0.0 255.255.255.255`
*   **Keyword Method (Preferred):**
    `R1(config)# access-list 11 permit any`
