## **2. Scan Methodologies**

### **Intrusiveness**
*   **Intrusive Scan:** Tries to **exploit** vulnerabilities. *Risk:* May crash the target system.
*   **Non-intrusive Scan:** Identifies vulnerabilities without exploitation. Safe for the target.

### **Access Level (Credentialed)**
| Type | Description | Pros/Cons |
| :--- | :--- | :--- |
| **Credentialed** | Uses valid usernames/passwords to log in. | âœ… More data, fewer errors.<br>âœ… Detects internal config issues. |
| **Non-Credentialed** | Scans from the "outside" (Hacker's view). | âŒ Less deep.<br>âŒ More false positives. |

### **Accuracy Terms**
*   **False Positive:** Scanner reports a vulnerability where **none exists**.
*   **False Negative:** Scanner **fails** to report an existing vulnerability.
> *Note: Credentialed scans significantly reduce both false positives and negatives.*

## **4. ğŸ¤– Security Automation (SIEM vs. SOAR)**

Automation is required to handle the massive volume of log data generated daily.

### **SIEM (Security Information and Event Management)**
*   **Core Function:** **Aggregates logs** from various sources (firewalls, servers) and combines similar events to reduce data load.
*   **Goal:** Identifies deviations from the norm (anomalies) and patterns.
*   **Key Tasks:**
    *   Identify internal/external threats.
    *   Monitor resource usage.
    *   Compliance reporting.
    *   Support incident response.

### **SOAR (Security Orchestration, Automation, and Response)**
*   **Core Function:** Collects threat data and **responds** to low-level events **without human intervention**.
*   **Key Tasks:**
    *   Threat & Vulnerability Management.
    *   Security Incident Response.
    *   **Automating** security operations.

> [!abstract] **Summary Difference**
> *   **SIEM** = **Detection & Logging** (Collecting data, finding the needle in the haystack).
> *   **SOAR** = **Action & Response** (Automatically fixing the issue or blocking the threat).
