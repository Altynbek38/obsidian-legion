## **1. Security Assessments & Scanners**

**Vulnerability Scanners** assess systems for weaknesses.
*   **Examples:** Nessus, Retina, Core Impact, GFI LanGuard.
*   **Types:** Network, Application, Web Application.

### **Scan Methodologies**
| Type | Description |
| :--- | :--- |
| **Intrusive** | Tries to **exploit** vulnerabilities; may crash the target. |
| **Credentialed** | Uses valid login info; harvests deep data; fewer false positives. |

### **Essential CLI Tools**
*   **Network:** `ipconfig`, `ping`, `arp`, `tracert`, `netstat`.
*   **DNS/NetBIOS:** `nslookup`, `nbtstat`.
*   **Scanning/Hacking:** `nmap`, `netcat`, `hping`.

### **Automation: SIEM vs. SOAR**
*   **SIEM:** Uses log collectors to **aggregate** and **analyze** data from multiple sources.
*   **SOAR:** Collects threat data and **responds** automatically (no human intervention).

## **4. Penetration Testing**
**Definition:** Simulating malicious attacks to identify weaknesses (**Ethical Hacking**).

> [!info] **The 4 Phases of Pen Testing**
> 1.  **Planning:** Defining rules and scope.
> 2.  **Discovery:** Reconnaissance (Passive & Active).
> 3.  **Attack:** Exploiting vulnerabilities to gain access.
> 4.  **Reporting:** Documenting findings and results.

### **Teams & Sniffing**
*   **ðŸ”´ Red Team:** Attackers (Offensive).
*   **ðŸ”µ Blue Team:** Defenders (Defensive).
*   **Packet Sniffers (Analyzers):** Intercept and log traffic.
    *   *Legitimate Use:* Troubleshooting bandwidth/network issues.
    *   *Malicious Use:* Stealing data/credentials.
